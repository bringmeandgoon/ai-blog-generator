## Target Audience

**Primary**: Developers who want to use or contribute to the platform
**Secondary**: Non-technical readers curious about AI-powered content pipelines

## Value Proposition

- **Knowledge**: Understand how a blog article goes from idea to publication in 5 steps
- **Emotional**: Make a complex automated pipeline feel friendly and approachable
- **Practical**: README illustration that explains the system at a glance

## Core Themes

| Theme | Narrative Potential | Visual Opportunity |
|-------|-------------------|-------------------|
| Human + AI collaboration | Developer submits idea, agents do the work | Developer at keyboard, robots in assembly line |
| Data transformation | Raw topic → polished article | Visual metamorphosis across pages |
| Quality assurance | Multiple checks before publish | Inspector robot with magnifying glass |

## Key Figures

**Developer (阿水)**: Submits topic, reviews article. Casual clothes, laptop.
**Search Bot**: Binoculars and magnifying glass, searching the web
**Architect Bot**: Hard hat, blueprint, drawing article structure
**Writer Bot**: Typewriter/pen, creative energy
**Check Bot**: Red pen, clipboard, stern expression
**Rewrite Bot**: Eraser and polish cloth, humanizing the text

## Story Arc (5 pages)

1. **Cover**: Platform logo, all agents assembled
2. **Input**: Developer types topic → frontend sends to pipeline
3. **Search + Architect**: Search Bot gathers data, Architect Bot draws outline
4. **Write + Check**: Writer Bot types article, Check Bot reviews with red pen
5. **Rewrite + Publish**: Rewrite Bot polishes, article goes live, Developer happy

## Content Signals

- Educational, how-to → clean ligne-claire style
- Pipeline/assembly metaphor → standard layout with clear panel progression
- Tech audience → minimal decoration, clear information flow
